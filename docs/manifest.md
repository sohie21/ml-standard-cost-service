# ML-манифест проекта

## Предыстория

В проекте рассматривается задача оценки себестоимости товара (`standard_cost`) по данным о транзакциях велосипедного магазина.

У заказчика есть исторические данные о товарах, брендах, категориях, размере товара, цене продажи и фактической себестоимости. На основе этих данных нужно построить сервис, который сможет оценивать `standard_cost` для новой транзакции.

## Ценностное предложение

Сервис помогает быстрее оценивать себестоимость товара и использовать эту оценку в аналитике маржинальности.

Потенциальная ценность:

- ускорение анализа транзакций;
- выявление товаров с необычной себестоимостью;
- поддержка решений по ценообразованию;
- снижение ручной работы аналитика.

## Цели

Ключевые цели проекта:

1. Построить воспроизводимый ML pipeline.
2. Обучить модель регрессии для предсказания `standard_cost`.
3. Развернуть модель как REST API.
4. Обеспечить контроль доступности сервиса через `/health`.
5. Обеспечить контроль метрик модели через `/metrics`.
6. Описать SLI/SLO на техническом, модельном и бизнес-уровнях.
7. Применить Metrics Driven Development и зафиксировать решение в ADR.

## Решение

Заявленный уровень зрелости ML-системы: **Level 2**.

В проекте реализована учебная MLOps-система со следующими компонентами:

- GitHub как source control.
- GitHub Actions как CI pipeline.
- Docker как способ упаковки приложения.
- Docker Compose или Docker CLI как способ запуска сервиса на VM.
- FastAPI как prediction service.
- MLflow для логирования экспериментов.
- Prefect для описания retraining pipeline.
- `/health` для проверки доступности сервиса.
- `/metrics` для публикации метрик модели и SLO.
- ADR для фиксации архитектурного решения по MDD.

## Осуществимость

Проект реализуется одним разработчиком в учебных целях.

Используемые инструменты:

- Python;
- pandas;
- scikit-learn;
- FastAPI;
- Docker;
- GitHub Actions;
- MLflow;
- Prefect;
- Yandex Cloud VM.

Выбранная архитектура достаточно простая для реализации одним студентом, но покрывает основные элементы production-like ML-системы.

## Данные

Источник данных: исторические транзакции велосипедного магазина.

Целевая переменная:

```text
standard_cost
```

Признаки:

```text
product_id
online_order
order_status
brand
product_line
product_class
product_size
list_price
transaction_month
transaction_dayofweek
```

Процесс подготовки данных:

1. Загрузка CSV.
2. Приведение денежных значений к float.
3. Преобразование даты.
4. Извлечение календарных признаков.
5. Кодирование категориальных признаков.
6. Разделение на train/test.

## Метрики

Бизнес-метрика:

- средняя абсолютная ошибка оценки себестоимости товара.

ML-метрики:

- MAE;
- RMSE;
- R2;
- data drift.

Онлайн-метрики:

- healthcheck availability;
- latency p95;
- error rate;
- container health.

## Оценка качества модели

Основная метрика качества: `MAE`.

Дополнительные метрики:

- `RMSE`;
- `R2`.

Так как задача является регрессионной, классификационные метрики `AUC`, `Precision` и `Recall` не используются как основные. Для данного типа задачи они заменены на регрессионные метрики качества.

## Подбор модели

Подход к подбору модели итеративный:

1. Подготовка baseline.
2. Обучение `RandomForestRegressor`.
3. Оценка качества по `MAE`, `RMSE`, `R2`.
4. Сравнение с порогами SLO.
5. Сохранение модели только при приемлемом качестве.

## Инференс

Тип инференса: real-time inference.

Сервис предоставляет endpoint:

```text
POST /predict
```

Входные данные передаются в формате JSON.

Выходные данные:

```text
predicted_standard_cost
model_version
```

## Обратная связь

Источники обратной связи:

- фактическое значение `standard_cost`;
- ошибка между prediction и actual value;
- технические метрики API;
- drift входных признаков;
- бизнес-ошибки по сегментам товаров.

Если качество модели становится ниже SLO, запускается retraining pipeline.

MDD используется: **да**.
